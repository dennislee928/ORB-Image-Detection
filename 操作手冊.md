## 1. 系統簡介

「圖片對比檢測工具」（以下簡稱 ORB 系統）是一套圖片相似度比對軟體。您只需要指定一張「基準原圖」，再選擇一個包含多張圖片的資料夾，系統就會自動將資料夾裡的每一張圖片與基準圖進行比對，計算出相似度，並將結果以排名表格呈現。

本工具適合用於以下情境：

- 從大量圖片中找出與特定圖片相似的檔案
- 比對圖片是否為同一張圖的不同版本（含翻轉、裁切等情形）
- 整理重複或類似的圖片檔案

系統採用 ORB 與 AKAZE 兩種特徵比對演算法，能偵測圖片的關鍵特徵點進行比對，即使圖片經過縮放、翻轉，仍然可以正確辨識。

## 4. 操作步驟

以下以完整的操作流程說明如何使用本工具。

### 步驟一：啟動程式

雙擊執行檔，程式啟動後會出現主畫面。畫面上方有程式標題「圖片對比檢測工具」，下方有三個按鈕：

- **「1. 選擇基準原圖」**：用來選擇您要比對的基準圖片
- **「2. 選擇比對資料夾」**：用來選擇包含待比對圖片的資料夾
- **「3. 執行檢索」**：開始進行比對（需先完成前兩步才能點選）

畫面中間有一個紅色按鈕「查看特徵熱點圖（分析圖片細節對應）」，用於查看比對的視覺化結果。

畫面下方是進度條與結果表格。

### 步驟二：選擇基準原圖

1. 點選「**1. 選擇基準原圖**」按鈕。
2. 在彈出的檔案選擇視窗中，找到您要當作比對基準的圖片。
3. 選取圖片後，點選「開啟」。

### 步驟三：選擇比對資料夾

1. 點選「**2. 選擇比對資料夾**」按鈕。
2. 在彈出的資料夾選擇視窗中，選擇包含待比對圖片的資料夾。
3. 點選「選擇資料夾」確認。

> **說明**：比對資料夾裡可以放任意數量的圖片，系統會逐一跟基準圖比對。資料夾內如有非圖片檔案（例如 Word、PDF），系統會自動忽略，不影響運作。

> **提醒**：基準原圖與比對資料夾可以在不同的位置，不需要放在同一個資料夾裡。

### 步驟四：執行檢索

1. 當基準圖和比對資料夾都選好後，「**3. 執行檢索**」按鈕會變為可點選狀態（深綠色）。
2. 點選「**3. 執行檢索**」開始比對。
3. 畫面上的進度條會顯示目前處理進度。
4. 處理完畢後，會跳出提示訊息告知結果資料夾的位置。
<img width="632" height="332" alt="image" src="https://github.com/user-attachments/assets/f33cdbc8-066e-4830-9604-9582b5d67571" />

> **說明**：第一次對某個資料夾執行檢索時，系統需要先提取所有圖片的特徵，處理時間較長（依圖片數量與電腦效能而定）。之後再次對同一個資料夾執行檢索（即使換了基準圖），因為特徵已經快取過了，速度會快非常多。

### 步驟五：查看結果
<img width="632" height="332" alt="image" src="https://github.com/user-attachments/assets/6d388e85-14b7-45d2-a6f5-8a551a0f467e" />

比對完成後，結果會顯示在下方表格中，每一列代表一張比對圖片，欄位說明如下：

| 欄位名稱 | 說明 |
|----------|------|
| Rank | 排名（依信心度由高到低排序） |
| Filename | 圖片檔案名稱 |
| Inliers | 特徵匹配點數量（數字越大表示共同特徵越多） |
| Confidence | 信心度百分比（越高表示越相似） |
| Level | 相似等級（極高相似、高度相似、中度相似、無關） |
| Status | 是否已提取至結果資料夾（「已提取」或「-」） |

如果您想更深入了解兩張圖片之間的對應關係，可以使用「特徵熱點圖」功能：

1. 在結果表格中，**點選**您想查看的那一列（該列會變成藍色反白）。
2. 點選紅色按鈕「**查看特徵熱點圖（分析圖片細節對應）**」。
3. 系統會開啟一個新視窗，左邊顯示基準圖，右邊顯示比對圖的特徵熱點分布，兩者之間會以綠色線條標示出對應的特徵點。
4. 熱點圖以色溫顯示：紅色區域代表特徵密集（高度匹配），藍色區域代表特徵稀疏。
5. 觀看完畢後，**按鍵盤上任意鍵**即可關閉熱點圖視窗。

## 5. 各功能詳細說明

### 5.1 雙引擎比對技術

本工具同時使用兩種圖片特徵偵測演算法：

- **ORB（Oriented FAST and Rotated BRIEF）**：速度快，適合快速篩選。系統設定為偵測最多 5,000 個特徵點。
- **AKAZE（Accelerated-KAZE）**：精確度高，適合細節比對。

兩種演算法的匹配結果會合併計算，確保比對結果的準確性與全面性。

### 5.2 翻轉偵測

系統會自動對每張比對圖片進行四種方向的比對：

| 比對方式 | 說明 |
|----------|------|
| 原圖 | 不做任何翻轉，直接比對 |
| 水平翻轉 | 左右鏡像翻轉後比對 |
| 垂直翻轉 | 上下翻轉後比對 |
| 水平 + 垂直翻轉 | 同時左右鏡像加上下翻轉後比對（等同旋轉 180 度） |

系統會取四種方向中信心度最高的結果作為該圖片的最終比對結果。這表示即使圖片被翻轉過，系統仍然能正確辨識出來。

### 5.3 多重 RANSAC 模型偵測

系統採用多重 RANSAC（Random Sample Consensus）演算法，最多可偵測 3 個幾何變換模型。這表示即使圖片中只有部分區域相似（例如裁切、拼貼），系統仍然有能力偵測到。

### 5.4 自動提取結果

信心度達到 30% 以上的圖片會被自動複製到結果資料夾。結果資料夾的命名規則為：

```
V83_Result_[基準圖檔名]
```

例如，基準圖檔名為「sample.jpg」，則結果資料夾名稱為「V83_Result_sample」。

此資料夾會建立在比對資料夾內部。

### 5.5 中文檔名支援

本工具完整支援中文檔名的圖片，包含：

- 基準圖檔名可以是中文
- 比對資料夾路徑可以包含中文
- 比對圖片的檔名可以是中文
- 結果表格中的檔名會正確顯示中文

### 5.6 高 DPI 螢幕支援

本工具已針對 Windows 高 DPI 螢幕（例如 4K 螢幕、高解析度筆電）進行最佳化，介面文字與元件不會出現模糊的情形。

## 6. 結果判讀

### 相似等級說明

| 等級 | 信心度範圍 | 說明 | 是否自動提取 |
|------|-----------|------|-------------|
| 極高相似 | 大於 85% | 幾乎可確定為同一張圖片或僅有微小差異 | 是 |
| 高度相似 | 大於 60%，至多 85% | 圖片有大量共同特徵，很可能來自相同來源 | 是 |
| 中度相似 | 大於 30%，至多 60% | 圖片有部分相似之處，可能有局部重疊或類似元素 | 是 |
| 無關 | 30% 以下 | 圖片之間沒有明顯的關聯性 | 否 |

### Inliers（特徵匹配點數）

Inliers 代表兩張圖片之間成功匹配的特徵點數量。數字越大，表示兩張圖片共享的視覺特徵越多。一般來說：

- 100 以上：兩張圖片有非常強的關聯
- 30 ~ 100：有一定程度的相似
- 30 以下：關聯性較低

### 信心度（Confidence）

信心度是一個百分比數值，綜合考量特徵匹配點數量與基準圖特徵點總數的比例。信心度越高，表示兩張圖片越相似。

## 7. 快取機制說明

為了提升效能，本工具內建智慧型快取機制：

### 運作方式

1. **首次分析**：第一次對某個資料夾執行檢索時，系統會提取資料夾內所有圖片的特徵，並將結果儲存為快取檔案 `.orb_cache.dat`，存放在比對資料夾內。
2. **後續使用**：之後再次對同一個資料夾執行檢索時（即使關掉程式重新開啟），系統會自動讀取快取，跳過特徵提取步驟，大幅縮短處理時間。
3. **增量更新**：如果資料夾中新增了圖片，系統只會針對新增的圖片提取特徵，不會重新處理已快取的圖片。
4. **換基準圖**：更換基準原圖時，不需要重新提取資料夾特徵，因為比對資料夾的特徵已經快取好了，只需要重新比對即可。

### 快取檔案

- 檔名：`.orb_cache.dat`
- 位置：比對資料夾內
- 屬性：此檔案以「.」開頭，在檔案總管中預設可能不會顯示（視您的資料夾設定而定）
- 大小：依資料夾內圖片數量而定，通常為數 MB 至數十 MB

> **提醒**：如果您刪除了 `.orb_cache.dat` 檔案，下次執行檢索時系統會重新提取所有圖片特徵，處理時間會較長，但不影響比對結果的正確性。

## 8. 常見問題 FAQ

### Q1：程式開啟時跳出「Windows 已保護您的電腦」怎麼辦？

**A**：這是 Windows SmartScreen 的安全警告，因為程式不是從已知的軟體發行商下載的。請點選「其他資訊」，再點選「仍要執行」即可。這不影響程式的安全性。

### Q2：第一次分析資料夾很慢，是正常的嗎？

**A**：是的，這是正常的。第一次分析時，系統需要逐一提取每張圖片的特徵（包含四種翻轉方向），所以會比較花時間。但是特徵提取完成後會儲存快取，之後再分析同一個資料夾就會非常快。

### Q3：資料夾裡有幾百張甚至上千張圖片，可以嗎？

**A**：可以。系統能處理大量圖片，只是第一次提取特徵時會花較多時間。建議耐心等待進度條跑完。處理完畢後因為有快取機制，後續操作都會很快速。

### Q4：換基準圖後需要重新等很久嗎？

**A**：不需要。換基準圖時，系統不需要重新提取比對資料夾的特徵（已經快取了），只需要對基準圖做特徵提取，然後進行比對運算，速度會快非常多。

### Q5：我修改了比對資料夾裡的圖片（例如覆蓋同名檔案），系統會知道嗎？

**A**：系統目前是以「檔名」判斷是否已快取。如果您用不同的圖片覆蓋了原本的檔案（但檔名相同），系統不會自動偵測到。建議在這種情況下，手動刪除資料夾內的 `.orb_cache.dat` 檔案，讓系統重新提取特徵。

### Q6：比對結果裡的「已提取」是什麼意思？

**A**：信心度達到 30% 以上的圖片會被系統自動複製一份到結果資料夾（V83_Result_基準圖名稱）。表格中 Status 欄位顯示「已提取」表示該圖片已被複製，顯示「-」表示信心度未達標準，不會被複製。

### Q7：熱點圖視窗開啟後如何關閉？

**A**：按鍵盤上的**任意鍵**即可關閉熱點圖視窗。請不要直接點視窗的「X」按鈕關閉，建議使用鍵盤按鍵關閉以確保程式正常運作。

### Q8：可以比對 GIF、BMP 或 TIFF 格式的圖片嗎？

**A**：目前僅支援 `.jpg`、`.jpeg`、`.png`、`.webp` 四種格式。如需比對其他格式的圖片，請先使用圖片編輯工具（如小畫家）將圖片轉存為上述支援的格式。

### Q9：為什麼有些圖片的信心度是 0%？

**A**：可能的原因包含：
- 圖片內容與基準圖完全無關
- 圖片太小或太模糊，導致無法提取到足夠的特徵點
- 圖片已損壞或格式異常

### Q10：結果資料夾已經存在時會怎樣？

**A**：如果結果資料夾已經存在，系統會直接將符合條件的圖片複製進去。如有同名檔案會直接覆蓋。建議在重新執行前，先備份或清空舊的結果資料夾。

### Q11：程式執行期間可以做其他事情嗎？

**A**：可以，但建議不要最小化程式視窗，以便觀察處理進度。在特徵提取與比對過程中，程式會持續更新進度條。

### Q12：程式需要網路連線嗎？

**A**：不需要。所有運算皆在本機電腦上執行，不會上傳任何圖片或資料。

## 9. 注意事項

### 使用前注意

1. **圖片格式**：請確認比對的圖片為支援的格式（.jpg、.jpeg、.png、.webp），其他格式的檔案會被自動忽略。
2. **資料夾路徑**：基準圖和比對資料夾可以放在不同的位置，不需要在同一個資料夾裡。
3. **磁碟空間**：請確保比對資料夾所在的磁碟有足夠的空間，用於存放快取檔案（.orb_cache.dat）和結果資料夾。

### 使用中注意

4. **耐心等候**：第一次分析大量圖片時，請耐心等待進度條完成，不要強制關閉程式，否則快取可能不完整。
5. **選取圖片再看熱點圖**：使用「查看特徵熱點圖」功能前，請先在表格中選取一列。如果沒有選取，系統會跳出提示訊息。
6. **熱點圖關閉方式**：熱點圖視窗請用鍵盤任意鍵關閉。

### 使用後注意

7. **結果資料夾**：比對結果會自動複製到「V83_Result_[基準圖名稱]」資料夾中。請到比對資料夾內查看。
8. **快取管理**：如果比對資料夾中的圖片有異動（刪除或更換同名檔案），建議刪除 `.orb_cache.dat` 讓系統重新建立快取。
9. **快取檔案位置**：`.orb_cache.dat` 儲存在比對資料夾內，如果您移動或複製比對資料夾，快取檔案會跟著移動，仍然有效。

### 效能建議

10. **建議圖片數量**：單一資料夾建議不超過數千張圖片。圖片數量過多時，首次特徵提取會需要較長時間。
11. **記憶體使用**：比對大量圖片時，程式會佔用較多記憶體。如果電腦記憶體不足（4GB 以下），建議關閉其他不必要的程式。
